# JSOS local Rokid CXR-L artifact

`client-l-1.0.1-jsos-stripped.aar` is a JSOS-built compatibility artifact derived from Rokid's public Maven artifact:

`com.rokid.cxr:client-l:1.0.1`

JSOS Core also depends on:

`com.rokid.cxr:client-m:1.2.1`

The original `client-l:1.0.1` artifact contains shared CXR bridge classes and native libraries that are also present in `client-m:1.2.1`. Including both official AARs directly causes duplicate class/native library build failures.

The JSOS-stripped AAR keeps the CXR-L link, authorization, media stream service, and callback classes needed for the Hi Rokid / CXR-L HUD deployment flow. It removes only the duplicate classes and native libraries already supplied by `client-m:1.2.1`.

The CXR-L deployment approach was implemented after reviewing public Hi Rokid / CXR-L workflow references from Anezium's Rokid-APKs and OverlayRec repositories, with permission from Anezium to reference that work. This checked-in AAR itself was generated by JSOS from Rokid's public Maven artifact rather than copied from those repositories.
